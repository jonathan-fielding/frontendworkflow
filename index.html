Steps of a frontend workflow

When developing a website there is a typical workflow we follow:

  • Scaffold
  • Abstract
  • Iterate
  • Build
  • Deploy

To support this there are many tools, this website aims to highlight which tools fit in where.

Scaffolding

When we start a new web development project, we quickly want to be able to jump into development however often we have to take the time to setup our project including download boilerplates, setup dependencies etc.

Scaffolding tools do this for us, so we can do what we enjoy most, starting to build our website.

Iteration

Every time we save our code and view it in our browser we are creating an iteration of the site, kind of like a version without a version number.

Iteration tools allow us to perform tasks every time we create one of these iterations, whether it be to add dependencies to a site or run tasks associated with creating the new iteration.
